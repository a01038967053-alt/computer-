## 3. 프로젝트 기획 의도
* **파편화된 가격 정보 통합**: 소비자가 조립 PC를 구성할 때 다나와, 컴퓨존, 네이버 쇼핑, 쿠팡 등 다양한 쇼핑몰의 가격과 배송 혜택을 각각 검색해야 하는 번거로움을 해결하고자 함.
* **실시간 가격 데이터와 추이 분석 제공**: 단순 고정 시세가 아닌, 최근 4주간의 가격 동향 차트를 시각적으로 제공하여 소비자가 '지금 구매하는 것이 적절한지' 합리적인 의사결정을 내릴 수 있도록 지원.
* **사용자 친화적 맞춤 견적 시스템**: 부품별 사양을 직관적으로 비교하고, 마음에 드는 제품을 가상의 장바구니(견적서)에 담아 총합산 가격과 쇼핑몰별 배송 조건을 한눈에 비교할 수 있는 통합 웹 어플리케이션 구축.

## 4. 전체 아키텍처

### 📋 메뉴 구성 (Information Architecture)
1. **메인 대시보드 (Dashboard)**
   * **CPU / GPU 핵심 비교 탭**: 주요 하이엔드/가성비 프로세서 스펙, 공식 기준 가격, 시장 트렌드 카드 노출.
   * **부품 상세 그리드**: RAM, SSD, 케이스, 파워, 쿨러 등 카테고리별 핵심 스펙 및 시정 단가 비교 리스트.
2. **실시간 다채널 가격 비교 모달 (Price Comparison Modal)**
   * **실시간 채널별 가격**: 다나와 최저가, 컴퓨존 실시간가, 네이버 쇼핑 멤버십가, 쿠팡 로켓배송가 자동 파싱 및 연동.
   * **최근 4주간 시세 변동 추이**: Recharts 기반 꺾은선 그래프 시각화.
   * **부품 설명서 및 혜택 요약**: 판매처별 배송 기간, 배송비 및 특이 사항 비교 정보 제공.
3. **가상 온라인 견적 빌더 (Estimate Builder)**
   * 부품 추가/삭제에 따른 실시간 총액 계산 및 가상의 견적서 구성.

### 🔄 Data Flow (데이터 흐름도)
```
[Client React UI] 
   │ 
   ▼ (1) GET /api/prices?partId=cpu-13400
[Express Proxy Server] 
   │ 
   ├─► [Memory Cache Check] (30초 TTL 캐시 히트 시 즉시 클라이언트 반환)
   │ 
   ├─► [External Target Request] (다나와/네이버/쿠팡 비동기 요청 - 1.2s 타임아웃 제한)
   ├─► [Compuzone Security Policy Boundary] (보안 및 봇 탐지 정책 대응 우회 미작동, 안전한 시세 알고리즘 연동)
   │ 
   ▼ (2) JSON Packet Response (VendorMalls Array)
[Client React UI] 
   │
   └─► Recharts (시세 그래프 렌더링)
   └─► URLSearchParams를 통한 실제 이커머스 다이렉트 검색 이동 (SearchProductKey 파라미터 적용)
```

### ⚙️ Work Flow (사용자 흐름도)
1. **탐색**: 사용자가 CPU/GPU 등 원하는 부품을 비교하고 상세 "가격 비교" 버튼을 클릭합니다.
2. **조회**: 서버 API가 트리거되어 다채널 최저가 단가와 최근 4주간의 동향 그래프를 팝업으로 렌더링합니다.
3. **이동**: 사용자가 "해당 쇼핑몰 상품 페이지로 이동" 클릭 시, 실제 쇼핑몰의 정상 검색 파라미터(`SearchProductKey` 등)가 적용된 정확한 딥링크 URL이 생성되어 새 탭으로 즉시 검색 결과로 랜딩됩니다.
4. **구성**: 견적서 담기 기능을 통해 조립 PC의 최종 예상 지출을 통합 관리합니다.

## 6. 사용 프롬프트 예시

### 💬 프롬프트 1 (실시간 다채널 API 연동 프롬프트)
> "React 클라이언트에서 직접 외부 쇼핑몰 주소를 호출하면 브라우저 CORS 보안 정책에 막힙니다. Express 백엔드에 `/api/prices` 프록시 라우트를 생성하고, 다나와, 네이버, 쿠팡, 컴퓨존의 가격을 안전하게 반환하는 API를 개발해 주세요. 안정성을 위해 각 채널별로 1.2초 타임아웃을 걸어주고, 메모리 캐시(TTL 30초)를 적용해 서버 부하를 최소화해 주세요."

### 💬 프롬프트 2 (실제 컴퓨존 검색 URL 연동 오류 수정 프롬프트)
> "컴퓨존으로 이동은 정상적으로 되지만 검색 결과가 빈 채로 열리는 문제가 있습니다. 컴퓨존의 실제 브라우저 주소창 검색 URL 패턴을 분석해 보니 `https://www.compuzone.co.kr/search/search.htm?Seargbl=1&hidden_Txt=&IsEventSearch=&SearchProductKey=검색어` 형태를 사용합니다. 임의의 `q`나 `query` 파라미터를 사용하지 말고, `URLSearchParams`를 활용해 `SearchProductKey` 파라미터에 안전하게 인코딩된 상품명을 전달해 새 창으로 열어주는 코드를 작성해 주세요."

## 7. 기술 스택
* **Frontend**: React (v18+), Vite, Tailwind CSS (스타일링), Lucide React (아이콘), Recharts (시세 변동 차트 시각화), Motion (인터랙션 애니메이션)
* **Backend**: Node.js, Express (v4), Tsx (TypeScript 실행 엔진), Esbuild (배포 서버 번들링)
* **Deployment**: Cloud Run (프로덕션 풀스택 컨테이너 구동), Vercel (정적 리소스 및 빌드 테스팅)

## 9. 프로젝트 소감 (PMI 형식)

### 👤 팀원 A (PM & 기획)
* **Plus (강점)**: AI 코딩 엔진과의 실시간 피드백을 통해 3일 걸릴 상세 UI 설계 및 기획안 구체화를 단 몇 시간 만에 마무리할 수 있었습니다. 특히 디자이너 없이도 아름다운 프리미엄 스페이스 그레이 다크 테마를 뽑아낸 점이 최고였습니다.
* **Minus (약점)**: 실제 쇼핑몰들의 크롤링 차단 및 보안 정책(403 Forbidden, 418 Bot Detection 등)이 매우 까다로워 외부 시세를 완전히 실시간 긁어오는 데 한계가 있었습니다. 
* **Interest (흥미로운 점)**: 크롤링 실패 시 무조건 에러 페이지를 보여주는 대신, 백엔드에서 사전에 구축해 둔 정적 시세 가이드라인을 기반으로 안전하게 기준가를 우회 제공(Fallback)하는 풀스택 아키텍처 아이디어를 학습하게 되어 매우 흥미로웠습니다.

### 👤 팀원 B (Full-Stack Developer)
* **Plus (강점)**: 이커머스 사이트별로 각각 다르게 사용하는 검색 URL 파라미터를 정합성 있게 분석해 낸 점이 기쁩니다. `SearchProductKey`와 같이 컴퓨존이 실제 사용하는 고유 파라미터를 브라우저 검증을 통해 찾아내 완벽한 원클릭 검색 연동을 구현했습니다.
* **Minus (약점)**: 복잡한 비동기 백엔드 프록시 개발 도중 캐시 갱신 주기(TTL) 설정과 외부 요청 타임아웃 세부 제어가 처음엔 엉켜 빈 화면이 뜨기도 했습니다.
* **Interest (흥미로운 점)**: React UI 단에서 `URLSearchParams`를 동적으로 빌드하고 이를 통해 이커머스 사이트에 정확히 검색 결과 페이지로 랜딩하게 만드는 정밀한 딥링크 라우팅 과정이 매우 유익하고 재밌는 경험이었습니다.
